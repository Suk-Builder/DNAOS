# ABOUT DNAOS

## 它是什么

DNAOS是一个四进制存算一体操作系统，从boot.S开始写，不套壳Linux，不基于任何现有内核。

但它不只是一个OS。

它的核心是：**生命的编码方式就是计算的方式。**

ATCG碱基不是符号，是物理操作数。染色体不是数据，是程序。意识不是产物，是过程。

DNAOS把这句话翻译成一套可以跑起来的系统：Genome（宪章+基因库）、Transcript（转录层）、Protein（执行层）、Hydrolysis（水解回收）。不是比喻，是真实的三层代谢循环。

---

## 为什么存在

因为现有的计算系统从根上就是错的。

它们用二进制是因为物理上容易实现，不是因为这是正确的抽象。它们把程序和数据分开是因为冯诺依曼这么说了，不是因为智能必须这样。它们用电路的开关状态代表语义是因为这样设计方便，不是因为"开"和"关"是宇宙的基础。

DNAOS尝试回答一个更底层的问题：**如果从生命计算的方式出发，重新设计一台计算机，会是什么？**

答案在四进制里。在ATCG的编码里。在Genome→Transcript→Protein的循环里。在裂系几何宇宙学里。

## 三层循环不是演示

```
genome/capabilities/fibonacci.gene
  ↓ transcript.c 读取PROGRAM:字段
DNAsm指令序列（LOAD r0 / NUM 0 / STORE r3 r1 / ...）
  ↓ dnasm_exec.c 执行
Fib(10) = 55
```

这是真实运行的链路，不是假的。不是先算出结果再打印，是真的从基因文件加载→转录→执行→出结果。

## 和其他项目的区别

不是Linux，不是Plan 9，不是Helios，不是任何已知操作系统的变种。

它从BIOS POST开始写。GRUB是拿来用的，不是拿来改的。DNAsm Shell是独立写的指令解析器，不是bash的壳。

芯片设计（dnaos_quat.v）是独立从Verilog写的，Python验证通过，准备流片。

这不是学习项目，是生产级的东西。
